The line has unsafe input parameter:

+ Why was this line marked?

ASIDE has detected a section of code that it suspects is vulnerable to a Cross-Site Scripting (XSS) attack. This specific warning asking for input validation is generated when ASIDE detects input from the user that has not been validated. In this line, the segment of code associated with reading input could be returning unsafe code which could then be used later in the application.

+ What could happen if this warning is ignored?

This line of code is not secure, and a malicious user could inject code and cause a Cross-site scripting (XSS) attack. XSS attacks are caused by injected malicious scripts in websites, which a user's browser then executes. Look at the example to see how XSS might work in webpage.

www.example.com/home.html?userName=asidedeveloper

A user could instead write something like:

www.example.com/home.html?userName=<script>alert(document.cookie)</script>

This example would just pop up the cookie, but in reality an injected script could have much more serious consequences for the end user.

+ How do you fix it?

Look back at your code. In this line, the segment of code associated with reading input should be validated before being used in the application. There are options below the warning to generate code to ensure the user input is appropiate for its purpose in the application. This prevents any injected code from being saved or used within the application.

For example for the following code snippet - "int newBalance = Integer.valueOf(request.getParameter("NewBalance"));" , the ASIDE will generate the following code:

int newBalance = Integer.valueOf(ESAPI.validator().getValidInput("Validation Context", request.getParameter("NewBalance"), "SafeString",200, false))

int newBalance = Integer.valueOf(ESAPI.validator().getValidInput("Validation Context", request.getParameter("NewBalance"), "CreditCard",200, false))

int newBalance = Integer.valueOf(ESAPI.validator().getValidInput("Validation Context", request.getParameter("NewBalance"), "Email",200, false))

int newBalance = Integer.valueOf(ESAPI.validator().getValidInput("Validation Context", request.getParameter("NewBalance"), "HTTPParameterValue",200, false))

int newBalance = Integer.valueOf(ESAPI.validator().getValidInput("Validation Context", request.getParameter("NewBalance"), "SSN",200, false))

int newBalance = Integer.valueOf(ESAPI.validator().getValidInput("Validation Context", request.getParameter("NewBalance"), "URL",200, false))
