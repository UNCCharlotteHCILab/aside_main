The line has possible vulnerable data output to user:

+ Why was this line marked?

ASIDE has detected a section of code that it suspects is vulnerable to a Cross-Site Scripting (XSS) attack. This specific warning asking for output encoding is generated when ASIDE detects output to the user that has not been encoded. In this line, the segment outputting data to the user could be returning unsafe code to the end user.

+ What could happen if this warning is ignored?

This line of code is not secure, and a malicious user could inject code and cause a Cross-site scripting (XSS) attack. XSS attacks are caused by injected malicious scripts in websites, which a user's browser then executes. Look at the example to see how XSS might work in webpage.

www.example.com/home.html?userName=asidedeveloper

A user could instead write something like:

www.example.com/home.html?userName=<script>alert(document.cookie)</script>

This example would just pop up the cookie, but in reality an injected script could have much more serious consequences for the end user.

+ How do you fix it?

Look back at your code. In this line, currentUser.getUsername() should be encoded before outputing to the user. There are options below the warning to generate code to encode the output to be used for URLs or HTML. This prevents any injected code from being run.

For example for the following code snippet - "out.println("Successfully logged in " + currentUser.getUsername());", the ASIDE will generate the following code:

out.println("Successfully logged in " + ESAPI.encoder().encodeForHTML(currentUser.getUsername()));
